# BLUEDOT 사업화 데이터 수급 가이드

현재 화면의 미지원 항목(도보권/노후도/리뷰/주차)을 실제 사업 운영 가능한 데이터 파이프라인으로 전환하기 위한 실행 가이드입니다.

## 1) 항목별 데이터 소스(실무용)

| 기능 | 1순위 소스 | 확보 경로 | 비고 |
|---|---|---|---|
| 자동차사고·자동차보험 수요 | `data/TAAS_전국_사고건수_최종마스터.xlsx` (시군구) | 도로교통공단 TAAS 통계 다운로드 후 월/분기 적재 | 현재 코드 연동 완료 |
| 경쟁 병원 개원연차 | HIRA `getHospBasisList`의 `estbDd` + (선택) 개폐업 API | 공공데이터포털 HIRA API 활용신청 | `estbDd`는 즉시 사용 가능 |
| 경쟁 병원 폐업이력 | `yadmOpCloInfoService2` (개폐업 이력) | 공공데이터포털 승인 후 호출 | 승인 반영/호출 조건 확인 필요(403 가능) |
| 리뷰 평점/불만 키워드 | 네이버 플레이스, 카카오맵 (정책준수) | 공식 제휴/API 또는 계약 기반 수급 | 무단 크롤링 금지 |
| 주차 거리/도보시간 | 공공데이터포털 주차장 + 보행 경로 API | 데이터포털 + 카카오/티맵 경로 API | 좌표 정합성 중요 |
| 시간대 유동(진료시간 추천) | 서울 생활인구 API + 통신 데이터(민간) | 서울 열린데이터광장 + 민간 계약 | 공공 데이터로 MVP 가능 |

## 2) 지금 바로 가능한 최소 사업화 스택 (MVP)

1. **사고수요**: TAAS 시군구 건수(이미 반영됨)  
2. **노후도**: HIRA `estbDd` 기반 개원연차(이미 반영됨)  
3. **주차**: 공영주차장 좌표(공공) + 거리 계산(Haversine)  
4. **도보 10분권**: 카카오/티맵 보행 API 계약 전까지는 반경 근사치 유지(법적 문구 명시)  
5. **리뷰**: 미지원 상태 유지 + 제휴 완료 후 오픈

## 3) 소스별 필수 컬럼(개발 연동 체크리스트)

### A. TAAS
- 필수: `지역명`, `사고건수`
- 권장: `연도`, `월`(시계열 확장)
- 현재 로직: `부산_해운대구` -> `부산 해운대구` 정규화

### B. HIRA 병원기초
- 필수: `ykiho`, `yadmNm`, `XPos`, `YPos`, `estbDd`
- 활용: 개원연차 = `현재연도 - estbDd[:4]`

### C. 주차장
- 필수: `주차장명`, `위도`, `경도`, `총주차면`, `운영시간`, `요금`
- 활용: 후보지 좌표와 최근접 주차장 거리/도보시간

### D. 리뷰
- 필수: `평점`, `리뷰본문`, `작성일`
- 활용: 월평균 평점, 키워드(친절/대기/가격) 빈도

## 4) 데이터 품질 규칙 (허위정보 방지)

- 실데이터 미수급 시 **숫자 생성 금지** (현재 코드 반영)
- 중복 지역은 `sum` 대신 `max`/정책 기준으로 처리 (TAAS 반영)
- 날짜 파싱 실패 시 `미지원` 표기
- 출처 라벨은 UI 비노출 상태이므로, 내부 로그/문서로만 관리

## 5) 운영 절차 (권장)

1. 월 1회 수집 스케줄 실행 (TAAS/HIRA/주차)
2. `data/` raw 보관 + `data/curated/` 정제본 생성
3. 배포 전 데이터 QA:
   - 지역 매칭 실패율 < 1%
   - 중복률/결측률 보고
4. 결과 리포트 샘플 20개 수기 검증

## 6) 법무/라이선스 체크

- 공공데이터: 이용허락 범위(상업적 이용/출처표시 의무) 확인
- 플랫폼 리뷰: 약관 위반 수집 금지, 반드시 정책 준수
- 개인정보: 원문 저장 시 비식별/보관기간 정책 필요

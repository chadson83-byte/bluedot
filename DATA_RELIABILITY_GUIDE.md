# BLUEDOT 데이터 신뢰성 개선 가이드

본 문서는 입지 분석 결과의 **신뢰성을 높이기 위한** 데이터 수집·변환 절차를 안내합니다.

---

## 1. 현재 구조와 신뢰도 이슈

### 1.1 데이터 흐름

```
[원본 데이터] → [변환 스크립트] → bluedot_master_v*.csv → main.py API
```

| 데이터 | 현재 소스 | 신뢰도 이슈 |
|--------|-----------|-------------|
| 의료기관 수/의사 수 | 1.병원정보서비스 (부산만 가능성) | 전국 데이터 필요 |
| 인구 | make_v3: `hosp_count*1200+5000` 추정 | **통계청 실데이터 필수** |
| 젊은층 비중 | `0.25 + subway*0.01` 추정 | 통계청 연령별 인구 필요 |
| 지하철 | 부산교통공사만 | 서울·대구·대전·광주 등 전국 필요 |
| 버스 | tl_bus_station_info | 전국 커버리지 확인 필요 |
| 상가 인프라 | 2_상가정보 CSV | 지역별 파일 추가 필요 |
| 경쟁도(docs_per_10k) | 전체 의사 수 기준 | **진료과목별 분리**로 정확도 상승 |

### 1.2 핵심 개선 포인트

1. **전국 의료기관 변환**  
   - 공공데이터포털 「병원정보서비스」 **전국** 엑셀 다운로드  
   - `1.병원정보서비스(2025.12.).xlsx`를 전국 버전으로 교체

2. **인구 실데이터 연동**  
   - 통계청 SGIS 또는 KOSIS에서 행정동별 인구·연령별 비중 확보  
   - `data/1_raw/행정동별_인구_실데이터.csv` 형식으로 저장

3. **전국 대중교통**  
   - 서울·부산·대구·대전·광주 등 도시철도 좌표 CSV 준비  
   - `scripts/make_master_nationwide.py` 설정에 경로 추가

4. **진료과목별 경쟁도**  
   - `5.의료기관별상세정보서비스_03_진료과목정보` 활용  
   - `docs_치과`, `docs_소아과` 등 과목별 의사 수 집계

## 2. 전국 마스터 데이터 생성 (권장 절차)

### 2.1 사전 준비

1. **병원정보 전국 버전**  
   - [공공데이터포털](https://www.data.go.kr) → 「병원정보서비스」 검색  
   - 전국 데이터 다운로드 → `data/1.병원정보서비스(2025.12.).xlsx`로 저장

2. **진료과목정보**  
   - `data/5.의료기관별상세정보서비스_03_진료과목정보 2025.12..xlsx`  
   - 이미 있다면 그대로 사용

3. **지하철역 좌표 (선택)**  
   - 공공데이터포털에서 지역별 도시철도역 좌표 CSV 다운로드  
   - 예: 서울교통공사, 대구도시철도, 대전도시철도, 광주도시철도  
   - `data/1_raw/`에 저장 후 `scripts/make_master_nationwide.py`의 `subway_files`에 경로 추가

4. **인구 실데이터 (최우선 권장)**  
   - [통계청 SGIS](https://sgis.kostat.go.kr) 또는 [KOSIS](https://kosis.kr)  
   - 행정동(읍면동) 단위 총인구, 연령별 비중 추출  
   - 예시 컬럼: `행정동명`, `총인구`, `젊은층비중`  
   - `data/1_raw/행정동별_인구_실데이터.csv`로 저장

### 2.2 마스터 데이터 생성 실행

## 3. 인구 실데이터 CSV 형식

`행정동별_인구_실데이터.csv` 예시:

```csv
행정구역(동읍면)별,총인구 (명),젊은층_비중
부산광역시 해운대구 우동,45000,0.32
부산광역시 해운대구 중동,32000,0.28
서울특별시 강남구 역삼동,28000,0.45
...
```

- `행정구역(동읍면)별`: `시도 + 시군구 + 읍면동` 형식 (예: "부산광역시 해운대구 우동")  
- `총인구 (명)`: 해당 행정동 총 인구  
- `젊은층_비중`: 20~30대 비율 (0~1, 예: 0.32 = 32%)

## 4. make_v6.py와 make_master_nationwide.py 비교

| 항목 | make_v6.py | make_master_nationwide.py |
|------|------------|---------------------------|
| 입력 | bluedot_master_v5 필요 | V5 불필요, 원본부터 생성 |
| 의료기관 | V5 의존 | 1.병원정보서비스 직접 사용 |
| 인구 | V5 의존 | 실데이터 CSV 또는 보수적 추정 |
| 지하철 | V5 의존 | 다중 CSV 지원(전국) |
| 진료과목별 | 없음 | docs_치과, docs_소아과 등 |
| 상가 | 2_상가정보 | 동일 |

**권장**: 신규 구축 시 `make_master_nationwide.py` 사용.
